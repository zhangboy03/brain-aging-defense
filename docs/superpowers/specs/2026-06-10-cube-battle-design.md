# 魔方大战（找不同抢答）设计

日期：2026-06-10
状态：按用户指示「研究规则后直接实现」，spec 记录设计决策

## 1. 玩法规则（来自用户提供的题目卡）

- 大屏并排显示**两张魔方堆叠马赛克图**（同尺寸彩色方格阵），两图**只有 1 个色块不同**。
- 双方队伍（清华 / 北大）肉眼比对抢答：**答对 +1 分，答错 −1 分**。
- **先到 5 分的队伍获胜**（可配置）。
- 答错不换题：本题继续，直到有人答对或主持人公布答案。

## 2. 视觉搜索难度设计（研究结论 → 生成器旋钮）

视觉搜索文献的三个结论直接映射为出题参数：

1. **色块数量（set size）**：低效搜索下，找到时间随格子数线性上升。
   → 难度档调网格尺寸：初级 12×12、中级 20×20、高级 28×28（与参考图一致量级）。
2. **目标-干扰相似度（target–distractor similarity）**：差异色块若与原色**差异大**会
   "pop-out"（与格子数无关，秒出）；与原色**相似**则必须逐格扫描，最难。
   → 换色策略两档：`任意换色`（pop-out 概率高，热场用）/ `相似色换色`
   （红↔橙、黄↔白、蓝↔绿，正式题用，缺省）。
3. **干扰异质性**：六色随机马赛克天然高异质、无规律，搜索必然低效——保持纯随机即可，
   不需要做"图案"。

附加规则：差异点位置全网格均匀随机（不偏向中心），两图渲染像素级对齐（同一组件、
同一布局参数），保证比对公平。

## 3. 架构：复用 hexagon 的「声明式 state」模式

```
public/cube-battle/
  admin.html   控制台（电脑）：出题/抢答/判分/比分/公布答案
  index.html   大屏（iPad ×2 镜像）：双图渲染、抢答状态、揭晓高亮、比分
```

- 控制台持有唯一真相 `S`，每个动作 `publish()`：写 localStorage + BroadcastChannel
  （同设备回退）+ `Sync.console('cube-battle').pushState(S)`（跨设备）。
- 大屏 `Sync.display('cube-battle', render)` 纯渲染；`storage`/`bc` 双回退监听；
  Screen Wake Lock 防息屏；全屏按钮。复用 `public/sync.js`，不改后端
  （房间名是路径参数，`cube-battle` 房间天然隔离）。
- 无倒计时（原规则没有），state 流量极低，**只用 state、不用 event**，语义零损失。

## 4. State 形状

```js
{
  phase: 'idle'|'playing'|'locked'|'revealed'|'finished',
  round: 1,                 // 第几题（无上限，打到获胜为止）
  cols: 20, rows: 20,
  grid: '0314252…',         // rows*cols 个 0-5 色码（左图）
  diffIndex: 137,           // 右图唯一不同的格子下标
  diffColor: 4,             // 右图该格的色码
  team: 'thu'|null,         // 当前抢答锁定的队
  scores: {thu:0, pku:0}, teams: {thu:'清华', pku:'北大'},
  winScore: 5, winner: '', message: '…', flashAt: 0,
  reveal: false             // true 时两屏高亮 diffIndex
}
```

右图不单独存：渲染时把 `grid[diffIndex]` 换成 `diffColor`。28×28 = 784 字符，
单条 state < 2KB。

## 5. 回合流程

```
出新题 → playing（两图可见，双方搜索）
  抢答(Q/P 或按钮) → locked（大屏横幅显示哪队作答；选手口述/指出位置，主持人对照
                       控制台上标了 ◎ 的答案预览人工判定）
    判对 → +1 → revealed（两屏高亮差异块 + 闪光）→ 到达 winScore? finished : 下一题
    判错 → −1 → 回 playing（同一题继续）
  无人能找到 → 主持人「公布答案」→ revealed（不计分）→ 下一题
```

主持人控制台另有：比分手动 ±1 修正、改队名、重置比赛、获胜分数/难度档/换色策略设置。

## 6. 大屏布局

- 横屏双栏：左右两图各占约 44vw，居中分隔；顶部 HUD（两队比分 + 第几题），
  底部状态横幅（抢答中/某队作答/答对揭晓/获胜）。
- 色板用魔方标准六色（白/黄/红/橙/蓝/绿，带描边方格，还原参考图观感）。
- `revealed` 时差异格双屏同步打高亮环 + 放大脉冲，正确答案一目了然。
- 竖屏（备用）退化为上下两图。

## 7. 成功标准

- 控制台出题后，两块大屏 1 秒内显示同一对图；任意断线重连后自动恢复当前题与比分。
- 同一题两图除 diffIndex 外逐格相同（渲染自同一 grid 字符串，结构上保证）。
- 判对/判错/公布答案的比分与相位流转正确；到 winScore 自动判胜并全屏宣告。
- 后端不可达时同设备多标签页仍可用（localStorage + BroadcastChannel 回退）。

## 8. 非目标

- 不做选手端点击作答（现场是口头/手指大屏 + 主持人人工判定，与 hexagon 同模式）。
- 不做计时器、题库、多房间。
- 不改 server/app.py（通用中转已覆盖）。
